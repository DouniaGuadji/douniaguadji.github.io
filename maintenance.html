Maintenance prédictive turbines (démo synthétique)

Accueil Projet Design

Contexte

Secteur : énergie. Objectif : détecter précocement des pannes de turbines à gaz pour réduire les arrêts non planifiés.

Problème

Classifier l'état (panne vs normal) avec un rappel élevé sur la classe minoritaire.

Données (synthétiques)

Données générées à partir d'hypothèses inspirées d'un cas réel. Aucune donnée interne n'est exposée.

Approche

  • Modèles testés : k-NN, Random Forest, MLP (choix justifiés).
  • Prétraitement : normalisation, gestion déséquilibre (stratification).
  • Expérimentation : traçage des runs, validation croisée.

Résultats

  • F1 globale : 0.80–0.84
  • Rappel (classe panne) : 0.72–0.80

Industrialisation

  • Endpoint FastAPI /predict, Docker.
  • CI (tests & lint) + déploiement (environnement de démo).
  • Monitoring basique (latence, erreurs).
